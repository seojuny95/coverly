# 작업 로그

- 2026-07-24 — 백엔드 오류 응답을 서버 생성 요청 ID가 포함된 공통 계약으로 통일하고, CORS에서 `X-Request-ID`와 `Retry-After`를 읽을 수 있게 정리. QA 스트림은 구조화된 `error` 이벤트로 종료하며, 실패·연결 종료 시 질문 사용량을 정확히 한 번 환급하고 복구된 잔여 횟수를 클라이언트와 동기화한다. 서버 로그와 프론트 진단은 예외 원문·입력값 없이 허용된 메타데이터만 남긴다. 프론트엔드는 사용자 메시지와 개발자 진단을 분리하고 요청 timeout·취소·제한적 재시도를 공통 경계로 옮겼으며 서버의 `Retry-After`를 준수한다. Render 유휴 기동은 DB·세션 저장소까지 확인하는 `/ready`로 판단하고, PDF 업로드 크기는 multipart 파싱 전 ASGI 경계에서도 제한한다.
- 2026-07-24 — 보험 분석과 업로드 흐름을 전면 점검해 결정적 요약과 LLM 총평의 API·상태를 분리하고, 총평 재시도가 기존 분석 결과에 영향을 주지 않도록 정리. 총평 문장은 서버 슬롯과 구조화 사실로 생성한 뒤 안전 검증·1회 교정을 거치며, 보험료 요약 막대와 보장 분석은 총평 상태와 독립적으로 유지한다. 실손의료보험 중복과 그 밖의 실손형 중복을 별도 카드로 나누고, 같은 담보는 계약별 상세를 접어 한 항목으로 표시한다. 업로드는 포트폴리오당 최대 5개를 허용하되 서버 파싱 동시성·대기열을 별도 제한하고, 파일당 10MB·100쪽·추출 문자·표 셀 한도와 XML 적재 제한을 적용했다. 프론트엔드는 OpenAPI 생성 타입·오류 계약을 사용하고, 세션 만료·부분 실패·취소 rollback·재시도 상태를 분리했다. React Compiler를 활성화하고 프론트엔드 업로드 오케스트레이션과 보장 합계 테이블을 기능별 파일로 분리했다.
- 2026-07-23 — 포트폴리오 총평을 `POST /portfolio/overview`로 분리하고, `POST /portfolio/summary`는 결정적 분석 결과만 반환하도록 변경. 총평 실패·재시도·세션 만료 상태가 보장 합계와 다른 분석 화면을 지우지 않도록 프론트엔드 query/hook을 분리하고 오류 카피와 비동기 재시도 피드백을 보강. QA 실패 시 질문 사용량을 환급하고 원문 질문·클라이언트 오류가 로그에 남지 않도록 정리했으며, 보장 상세 펼침 애니메이션과 로컬 샘플 보험증권 4개를 추가했다.
- 2026-07-22 — 질문 재작성·범위 판단을 앞단에서 확정하던 counsel planner/composer 파이프라인을 제거하고, 결정적 사실 조회와 두 RAG를 11개 읽기 전용 도구로 제공하는 단일 QA agent로 통합. 외부 상담 API를 `POST /qa/stream` 하나로 정리하고 관련 런타임·평가·테스트·프론트엔드 계약을 `qa` 경계로 이동. 현재 QA 평가셋 70개 대화·89턴의 자동 평가는 61/89가 통과했다. 자동 실패 28턴을 사람이 모두 검수해 21턴을 통과, 7턴을 실패로 판정했으며 검수 반영 기준선은 82/89다(PR #58, #59).
- 2026-07-19 — 프론트엔드에 공용 UI 프리미티브 레이어(shadcn/ui: Card·Badge·Alert·Skeleton·Dialog·AlertDialog·Input·Label·RadioGroup)를 도입하고 기존 룩에 맞춰 커스터마이즈(리팩터, 화면 그대로). 큰 파일을 분리(`recommendation-cards`·`panel`은 폴더로, `screen`은 `useExpandedRows`/`useTabNavigation` 훅과 `InsuranceListPanel`로)하고, Q&A 로직을 `useQaChat`로, 업로드 오케스트레이션을 `handleSubmit` named-step + `upload-helpers`로 추출. 모달은 radix Dialog/AlertDialog로 전환. react-hooks 규칙을 error로 승격(PR #54).
