# UX 카피 가이드

Coverly의 사용자 대상 문구는 보험 용어를 최대한 쉽게 풀어 쓴다. 사용자는 보험증권 PDF와 보장 내용을 이미 어렵게 느낄 수 있으므로, 화면 문구는 기능 설명보다 사용자가 얻게 되는 결과를 먼저 말한다.

## 기본 원칙

- **해요체를 사용한다.** 딱딱한 명사형보다 사용자가 듣기 쉬운 문장으로 쓴다.
- **사용자 결과를 먼저 말한다.** 시스템 동작보다 사용자가 무엇을 볼 수 있는지 설명한다.
- **짧게 쓴다.** 한 문장은 한 가지 의미만 담고, 불필요한 수식어를 줄인다.
- **능동형으로 쓴다.** “표시됩니다”보다 “보여드려요”처럼 행동 주체가 분명한 문장을 우선한다.
- **어려운 보험·개발 용어를 피한다.** 꼭 필요한 용어는 화면 맥락 안에서 쉽게 이해되도록 쓴다.

## 선호 표현

| 피할 표현                                                   | 선호 표현                                            |
| ----------------------------------------------------------- | ---------------------------------------------------- |
| 보험증권 묶음 분석                                          | 내 보험 분석                                         |
| 여러 PDF를 올리면 분류별로 정리한 분석 화면으로 이동합니다. | 보험증권 PDF를 올리면 내 보험을 분류해서 보여드려요. |
| 파일을 끌어오거나 선택하세요                                | 보험증권 PDF를 올려주세요                            |
| PDF 선택                                                    | PDF 불러오기                                         |
| 분석 시작                                                   | 내 보험 분석하기                                     |
| 증권을 분석하고 있습니다.                                   | 보험을 정리하고 있어요.                              |
| 분석이 끝나면 결과 화면으로 이동합니다.                     | 끝나면 결과 화면으로 이동해요.                       |

## 버튼 문구

- 버튼은 누르면 일어나는 일을 동사로 쓴다.
- “확인”, “Submit”, “다음”처럼 결과가 모호한 문구는 피한다.
- 예: `내 보험 분석하기`, `PDF 불러오기`, `선택한 피보험자로 보기`

## 에러 문구

에러는 “무슨 일이 일어났는지”와 “어떻게 해결할 수 있는지”를 함께 말한다.

- 좋은 예: `피보험자를 확인할 수 없는 증권이 있습니다. 피보험자가 있는 증권만 분석할 수 있습니다.`
- 나쁜 예: `INVALID_INPUT`, `업로드 실패`, `문제가 발생했습니다.`

## 업로드 화면 카피 방향

업로드 화면은 사용자가 PDF를 올리면 어떤 결과를 얻는지 바로 이해해야 한다.

- 제목은 `내 보험 분석`처럼 사용자가 얻는 결과를 말한다.
- 설명은 `보험증권 PDF를 올리면 내 보험을 분류해서 보여드려요.`처럼 짧고 구체적으로 쓴다.
- 로딩 문구는 `보험을 정리하고 있어요.`처럼 기다리는 동안 무슨 일이 진행 중인지 알려준다.

## 색상 원칙

화면 색상은 단순하게 유지한다. 기본 팔레트는 세 가지 색상만 사용한다.

| 역할  | 색상      | 용도                               |
| ----- | --------- | ---------------------------------- |
| White | `#FFFFFF` | 기본 배경, 카드 배경               |
| Ink   | `#111827` | 제목, 본문, 주요 버튼              |
| Blue  | `#2563EB` | 선택 상태, 포커스, 강조, 진행 상태 |

연한 배경, 테두리, 비활성 상태는 위 색상의 투명도만 사용한다. 새로운 색상을 추가하기 전에 같은 의미를 기존 팔레트로 표현할 수 있는지 먼저 확인한다.

## Coverly 톤

쉽고, 짧고, 행동 중심으로 쓴다. Coverly는 보험 정보를 다루므로 과하게 가볍게 쓰지 않고, 사용자가 안심하고 이해할 수 있는 수준의 차분한 톤을 유지한다.
