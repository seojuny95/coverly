# UX 카피 가이드

Coverly의 사용자 대상 문구는 보험 용어를 쉽게 풀어 쓰되, 과하게 가볍지 않게 유지한다. 이 문서는 참고한 UX 라이팅 원칙을 Coverly에 맞게 다시 정리한 기준이다.

## 이 문서의 목적

- 화면 문구를 쓸 때 같은 기준으로 판단한다.
- 추상적인 톤 설명보다 바로 적용할 수 있는 규칙을 우선한다.
- Coverly의 보험 맥락과 프로젝트 원칙을 함께 반영한다.

## 1. 라이팅의 핵심 가치

문장을 평가할 때 다음 다섯 가지를 기준으로 본다.

- **Clear**: 한 번 읽고 바로 이해할 수 있어야 한다.
- **Concise**: 꼭 필요한 말만 남겨 짧게 쓴다.
- **Casual**: 딱딱한 업계 말투 대신 자연스럽게 쓴다.
- **Respectful**: 사용자를 몰아붙이거나 과장하지 않는다.
- **Emotional**: 정보만 전달하지 말고 사용자의 감정도 고려한다.

## 2. 기본 라이팅 원칙

### Predictable hint

- 다음 화면에서 무슨 일이 일어나는지 예측할 수 있게 쓴다.
- 사용자 이익만 강조하고, 바로 이어질 행동을 숨기지 않는다.

### Weed cutting

- 넣어도 빼도 뜻이 바뀌지 않는 단어를 지운다.
- 모바일에서는 읽기보다 스캔이 먼저라는 점을 전제로 쓴다.

### Remove empty sentences

- 제목을 반복하는 설명, 빈 공간을 채우기 위한 문장을 넣지 않는다.
- 한 화면 안에서 같은 말을 두 번 하지 않는다.

### Focus on key message

- 지금 꼭 알아야 하는 정보만 남긴다.
- 친절함을 이유로 맥락을 과하게 설명하지 않는다.

### Easy to speak

- 소리 내어 읽어도 자연스러운 문장으로 쓴다.
- 문어체, 한자어, 업계 용어를 줄인다.
- 가능하면 한 문장에는 한 메시지만 담는다.

### Suggest over force

- 행동을 강요하거나 손해 공포를 주지 않는다.
- 선택권이 사용자에게 있다는 느낌을 남긴다.

### Universal words

- 특정 업계, 세대, 집단만 이해하는 단어를 쓰지 않는다.
- 유행어, 은어, 밈보다 사실 전달을 우선한다.

### Find hidden emotion

- 문구를 읽는 순간 사용자가 느낄 감정을 고려한다.
- 특히 완료, 실패, 대기, 불안 상황에서는 정보보다 감정 대응이 먼저일 수 있다.

## 3. 문장 규칙

실제 UI 카피는 아래 규칙을 우선 적용한다.

### 해요체

- 제품 안의 문구는 기본적으로 해요체로 쓴다.
- 상태, 버튼, 안내, 에러를 가리지 않고 일관되게 유지한다.

예:

- `보험을 정리하고 있어요.`
- `PDF 파일만 업로드할 수 있어요.`

### 능동적 말하기

- 가능한 한 능동형으로 쓴다.
- `됐어요`보다 `했어요`, `표시됩니다`보다 `보여드려요`처럼 동작이 보이게 쓴다.
- 수동형이 더 짧고 명확한 특수한 경우에만 예외로 쓴다.

### 긍정적 말하기

- `안 돼요`, `불가능해요`, `없어요` 같은 부정형을 기계적으로 반복하지 않는다.
- 사용자가 할 수 있는 다음 행동을 중심으로 쓴다.

예:

- 나쁨: `지금은 혜택을 받을 수 없어요.`
- 좋음: `보험증권을 올리면 분석을 시작할 수 있어요.`

### 캐주얼한 경어

- `~시겠어요?`, `~께`, `여쭈다`, `계시다`처럼 과한 경어는 쓰지 않는다.
- 공손함보다 자연스러운 친절함을 우선한다.

예:

- `피보험자에게 알릴게요`
- `궁금한 내용을 묻고 답을 확인해요`

### `{명사} + {명사}` 줄이기

- 한자어 명사를 이어 붙인 표현보다 동사형 표현을 우선한다.
- 꼭 풀어쓰기 어렵다면 최소한 문장으로 풀어 쓴다.

예:

- `보험 분석 결과`보다 `보험을 분석한 결과`
- `피보험자 선택`보다 `피보험자를 선택해요`

### 다이얼로그 버튼 문구

- 닫기 성격의 왼쪽 버튼은 `취소`보다 `닫기`를 우선한다.
- 버튼은 역할이 아니라 결과를 드러내야 한다.

## 4. Coverly에 적용하는 추가 원칙

일반적인 라이팅 원칙만으로는 Coverly의 도메인 특성이 충분히 반영되지 않는다. 아래 기준을 함께 적용한다.

### 근거 없는 판단 금지

- 약관, 추출 데이터, 사용자 입력으로 확인되지 않으면 단정하지 않는다.
- `보장돼요`처럼 확정형 표현보다 `확인된 내용이에요`, `약관을 더 확인해야 해요`처럼 근거 수준이 드러나는 표현을 쓴다.

### 보험 용어는 풀어 쓴다

- 보험사 내부 용어, 약관 용어, 분류 체계는 가능한 한 쉬운 말로 바꾼다.
- 그대로 써야 하는 용어는 바로 이해할 수 있는 설명을 붙인다.

### 개인정보를 카피에 노출하지 않는다

- 이름, 증권번호, 주민번호 등 식별 정보는 꼭 필요할 때만 제한적으로 쓴다.
- 에러 문구나 빈 상태 문구에는 식별 정보를 넣지 않는다.

### 안심을 먼저 준다

- 업로드, 분석, 실패 상황에서 사용자 불안을 키우지 않는다.
- 무슨 일이 진행 중인지, 무엇을 할 수 있는지 차분하게 알려준다.

## 5. 화면별 작성 규칙

### 제목

- 화면의 기능명이 아니라 사용자가 얻게 될 결과를 먼저 말한다.

예:

- `내 보험을 한눈에 정리해요`
- `내 보험을 종류별로 정리했어요`
- `보장금 합계를 확인해요`
- `분석할 보험증권이 없어요`

### 설명 문구

- 제목을 반복하지 말고, 바로 다음 행동이나 기대 결과를 알려준다.
- 한 문장에는 한 가지 정보만 담는다.

예:

- `보험증권 PDF를 업로드하면 흩어진 보험 정보를 보기 쉽게 분석해드려요.`
- `업로드 화면에서 PDF를 선택하면 분석 결과가 여기에 표시돼요.`
- `손해보험은 종류별 보장금으로 따로 확인할 수 있어요.`

### 버튼 문구

- 버튼은 누르면 일어나는 행동을 동사로 쓴다.
- `확인`, `다음`, `Submit`처럼 결과가 흐린 표현은 피한다.
- 가능하면 `~하기` 또는 명확한 동사구로 쓴다.

예:

- `PDF 불러오기`
- `내 보험 분석하기`
- `선택한 피보험자로 보기`
- `보험증권 더 올리기`

### 로딩 문구

- 무엇을 처리 중인지 알려줘야 한다.
- 단순한 대기 문구보다 진행 상황을 드러내는 문구를 쓴다.

예:

- `보험을 정리하고 있어요.`
- `PDF 내용을 확인하고 있어요.`
- `분석 결과를 불러오는 중이에요.`

### 빈 상태 문구

- 무엇이 비어 있는지 먼저 말하고, 다음 행동을 이어서 제안한다.

예:

- `선택된 PDF가 없어요.`
- `분석할 보험증권이 없어요.`
- `업로드 화면에서 PDF를 선택하면 분석 결과가 여기에 표시돼요.`

### 에러 문구

- `무슨 일이 있었는지 + 사용자가 할 수 있는 일`을 함께 쓴다.
- 내부 코드명, 예외명, 개발 용어는 노출하지 않는다.
- 부정형만 반복하지 말고 해결 행동을 붙인다.

예:

- `피보험자를 확인할 수 없는 증권이 있어요. 피보험자가 확인된 증권만 분석할 수 있어요.`
- `PDF 파일만 업로드할 수 있어요.`
- `파일이 너무 커요. 최대 10MB까지 업로드할 수 있어요.`
- `업로드에 실패했어요. 잠시 후 다시 시도해주세요.`

### 선택 유도 문구

- 선택해야 하는 이유는 설명하되, 손해를 암시하지 않는다.
- 사용자의 결정을 대신하지 않는다.

예:

- `분석에 포함할 피보험자를 선택하세요.`
- `선택한 피보험자의 증권만 결과에 포함돼요.`

## 6. 선호 표현

| 피할 표현                              | 선호 표현                                      |
| -------------------------------------- | ---------------------------------------------- |
| 보험증권 묶음 분석                     | 내 보험 분석                                   |
| 파일을 끌어오거나 선택하세요           | 보험증권 PDF를 올려주세요                      |
| PDF 선택                               | PDF 불러오기                                   |
| 분석 시작                              | 내 보험 분석하기                               |
| 증권을 분석하고 있습니다               | 보험을 정리하고 있어요                         |
| 분석이 끝나면 결과 화면으로 이동합니다 | 끝나면 결과 화면으로 이동해요                  |
| 문제가 발생했습니다                    | 업로드에 실패했어요. 잠시 후 다시 시도해주세요 |
| 보험 분석 결과                         | 보험을 분석한 결과                             |
| 피보험자 선택                          | 피보험자를 선택해요                            |

## 7. 문장 점검 체크리스트

- 이 문장은 다음 화면이나 다음 행동을 예측하게 하는가?
- 지금 꼭 필요한 정보만 남겼는가?
- 말로 읽어도 자연스러운가?
- 해요체, 능동형, 긍정형을 지켰는가?
- 업계 용어와 한자어를 너무 많이 쓰지 않았는가?
- 사용자를 겁주거나 몰아붙이지 않는가?
- 근거 없는 판단이나 과장이 없는가?
